Lập Trình Hướng Đối Tượng (OOP) trong JavaScript

Lập Trình Hướng Đối Tượng (OOP) trong JavaScript Link to heading

Lập trình hướng đối tượng (OOP) là một phương pháp lập trình phổ biến giúp tổ chức mã nguồn hiệu quả hơn thông qua việc sử dụng các đối tượng. JavaScript hỗ trợ OOP và cung cấp các khái niệm như lớp (class), đối tượng (object), kế thừa (inheritance), đóng gói (encapsulation) và đa hình (polymorphism). Trong bài viết này, chúng ta sẽ khám phá các khái niệm này và cách sử dụng chúng trong JavaScript.

1. Đối Tượng (Object) Link to heading

Đối tượng trong JavaScript là một tập hợp các thuộc tính và phương thức. Mỗi thuộc tính có một tên và giá trị, và mỗi phương thức là một hàm được lưu trữ trong đối tượng.

Tạo Đối Tượng: Link to heading 

let person = {
    name: "John",
    age: 30,
    greet: function() {
        console.log("Hello, " + this.name);
    }
};

person.greet(); // In ra “Hello, John” Trong ví dụ trên, person là một đối tượng có ba thuộc tính (name, age, greet). Phương thức greet là một hàm được sử dụng để in ra lời chào.

2. Lớp (Class) trong JavaScript Link to heading

Lớp là một bản thiết kế (template) cho việc tạo ra các đối tượng. Nó mô tả các thuộc tính và phương thức mà các đối tượng của lớp đó sẽ có. JavaScript hỗ trợ cú pháp class kể từ ES6.

Tạo Lớp:

Sao chép mã
class Person {
    constructor(name, age) {
        this.name = name;
        this.age = age;
    }

    greet() {
        console.log("Hello, " + this.name);
    }
}

let person1 = new Person("John", 30);
let person2 = new Person("Jane", 25);

person1.greet(); // In ra "Hello, John"
person2.greet(); // In ra "Hello, Jane"
Lớp Person có một phương thức greet và một hàm khởi tạo constructor để thiết lập các thuộc tính name và age.

3. Kế Thừa (Inheritance) Link to heading

Kế thừa là một cơ chế trong OOP cho phép một lớp (lớp con) kế thừa các thuộc tính và phương thức của một lớp khác (lớp cha). JavaScript sử dụng từ khóa extends để thực hiện kế thừa.

Ví dụ về Kế Thừa:

Sao chép mã
class Animal {
    constructor(name) {
        this.name = name;
    }

    speak() {
        console.log(this.name + " makes a sound");
    }
}

class Dog extends Animal {
    speak() {
        console.log(this.name + " barks");
    }
}

let dog = new Dog("Buddy");
dog.speak(); // In ra "Buddy barks"
Trong ví dụ trên, lớp Dog kế thừa từ lớp Animal và ghi đè phương thức speak.

4. Đóng Gói (Encapsulation) Link to heading

Đóng gói là cơ chế ẩn các chi tiết triển khai của một đối tượng, chỉ cung cấp các phương thức cần thiết để tương tác với đối tượng đó. Trong JavaScript, chúng ta có thể sử dụng các thuộc tính và phương thức riêng (private) để đóng gói dữ liệu.

Ví dụ về Đóng Gói:

Sao chép mã
class Person {
    #name; // Thuộc tính private
    #age;

    constructor(name, age) {
        this.#name = name;
        this.#age = age;
    }

    getName() {
        return this.#name;
    }

    setName(name) {
        this.#name = name;
    }
}

let person = new Person("John", 30);
console.log(person.getName()); // In ra "John"
person.setName("Mike");
console.log(person.getName()); // In ra "Mike"

Ở đây, thuộc tính #name và #age là private và không thể truy cập trực tiếp từ bên ngoài lớp. Chúng ta sử dụng các phương thức getter và setter để thao tác với chúng.

5. Đa Hình (Polymorphism) Link to heading

Đa hình cho phép các đối tượng thuộc các lớp khác nhau có thể có các hành vi khác nhau với cùng một phương thức. JavaScript hỗ trợ đa hình thông qua kế thừa và ghi đè phương thức.

Ví dụ về Đa Hình:

Sao chép mã
class Animal {
    speak() {
        console.log("Animal makes a sound");
    }
}

class Dog extends Animal {
    speak() {
        console.log("Dog barks");
    }
}

class Cat extends Animal {
    speak() {
        console.log("Cat meows");
    }
}

let dog = new Dog();
let cat = new Cat();

dog.speak(); // In ra "Dog barks"
cat.speak(); // In ra "Cat meows"

Trong ví dụ trên, mặc dù cả Dog và Cat đều kế thừa từ Animal, nhưng mỗi lớp có cách triển khai riêng cho phương thức speak.
