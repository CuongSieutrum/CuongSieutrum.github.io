1. Xử lý Ngoại lệ trong Java (Exception Handling) Link to heading

Trong lập trình Java, ngoại lệ (exception) là những sự kiện không mong muốn xảy ra trong quá trình chạy chương trình, làm gián đoạn dòng chảy bình thường của chương trình. Để xử lý những tình huống này, Java cung cấp cơ chế xử lý ngoại lệ giúp chương trình tiếp tục thực thi một cách an toàn và hiệu quả.

1. Cấu trúc cơ bản của xử lý ngoại lệ Link to heading

Java sử dụng khối mã try-catch để xử lý ngoại lệ. Cấu trúc cơ bản như sau:

Sao chép mã
try {
    // Mã có thể gây ra ngoại lệ
} catch (ExceptionType e) {
    // Xử lý ngoại lệ
} finally {
    // Mã này luôn được thực thi, dù có ngoại lệ hay không
}

try block: Đoạn mã có thể gây ra ngoại lệ. catch block: Bắt và xử lý ngoại lệ. finally block: Làm sạch các tài nguyên (nếu có), mã này luôn được thực thi dù có xảy ra ngoại lệ hay không. Ví dụ đơn giản

Sao chép mã
public class Example {
    public static void main(String[] args) {
        try {
            int[] numbers = new int[5];
            numbers[10] = 50;  // Đây sẽ gây ra ngoại lệ ArrayIndexOutOfBoundsException
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("Ngoại lệ: Lỗi chỉ mục mảng ngoài phạm vi!");
        } finally {
            System.out.println("Đoạn mã này sẽ luôn được thực thi.");
        }
    }
}

Giải thích:

Khối try chứa mã có thể gây ra ngoại lệ (ở đây là lỗi chỉ mục mảng). Khối catch sẽ bắt ngoại lệ ArrayIndexOutOfBoundsException và hiển thị thông báo lỗi. Khối finally sẽ luôn được thực thi, bất kể có xảy ra ngoại lệ hay không. Các loại ngoại lệ trong Java Checked Exceptions (Ngoại lệ đã được kiểm tra):

Là các ngoại lệ mà người lập trình phải xử lý hoặc khai báo trong phương thức bằng cách sử dụng từ khóa throws. Ví dụ: IOException, SQLException. Ví dụ:

Sao chép mã
public void readFile() throws IOException {
    FileReader file = new FileReader("file.txt");
}

2. Unchecked Exceptions (Ngoại lệ không kiểm tra): Link to heading

Là các ngoại lệ không bắt buộc phải xử lý hoặc khai báo. Chúng là các ngoại lệ kế thừa từ lớp RuntimeException. Ví dụ: NullPointerException, ArithmeticException, ArrayIndexOutOfBoundsException. Ví dụ:

Sao chép mã
public void divide() {
    int result = 10 / 0;  // Lỗi ArithmeticException (chia cho 0)
}

Nâng cao: Tạo Ngoại lệ tùy chỉnh Bạn có thể tạo ngoại lệ riêng của mình bằng cách kế thừa từ lớp Exception hoặc RuntimeException.

Sao chép mã
class InvalidAgeException extends Exception {
    public InvalidAgeException(String message) {
        super(message);
    }
}

public class Example {
    public static void checkAge(int age) throws InvalidAgeException {
        if (age < 18) {
            throw new InvalidAgeException("Tuổi không hợp lệ! Cần ít nhất 18 tuổi.");
        }
    }

    public static void main(String[] args) {
        try {
            checkAge(15);  // Gây ra ngoại lệ InvalidAgeException
        } catch (InvalidAgeException e) {
            System.out.println(e.getMessage());
        }
    }
}

Giải thích:

InvalidAgeException là một ngoại lệ tùy chỉnh, kế thừa từ Exception. Phương thức checkAge sẽ ném ngoại lệ nếu tuổi nhỏ hơn 18. Một số phương thức hữu ích trong lớp Throwable (lớp cha của Exception) getMessage(): Trả về thông báo chi tiết về ngoại lệ. printStackTrace(): In ra ngăn xếp gọi của ngoại lệ, giúp bạn xác định vị trí xảy ra lỗi. Ví dụ:

Sao chép mã
try {
    int[] numbers = new int[5];
    numbers[10] = 50;  // Lỗi ArrayIndexOutOfBoundsException
} catch (ArrayIndexOutOfBoundsException e) {
    e.printStackTrace();  // In thông tin chi tiết của ngoại lệ
}

Tổng kết Xử lý ngoại lệ giúp chương trình hoạt động ổn định hơn, tránh việc bị dừng đột ngột khi gặp lỗi. Java cung cấp cơ chế try-catch-finally để xử lý ngoại lệ, với khả năng quản lý ngoại lệ theo từng loại (checked và unchecked). Các ngoại lệ có thể được tùy chỉnh để phục vụ nhu cầu riêng của ứng dụng. Hy vọng rằng những kiến thức về xử lý ngoại lệ trong Java này sẽ giúp bạn viết mã an toàn và hiệu quả hơn.
