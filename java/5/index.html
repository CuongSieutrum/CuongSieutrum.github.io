Java Collections Framework

Java Collections Framework Link to heading

Java Collections Framework (JCF) là một bộ các lớp và giao diện được cung cấp trong Java để làm việc với các nhóm đối tượng. Nó cung cấp một cách tiếp cận thống nhất để lưu trữ và xử lý dữ liệu, giúp lập trình viên dễ dàng thao tác với các tập hợp đối tượng.

1. Các thành phần chính trong Collections Framework Link to heading

Java Collections Framework bao gồm các thành phần chính sau:

  • Collection Interface: Là giao diện cơ sở cho tất cả các kiểu collection trong Java.
  • List, Set, Queue, Map: Là các kiểu dữ liệu con của Collection (trừ Map, không phải là một con của Collection).
  • Implementations: Là các lớp cài đặt các giao diện của framework, ví dụ: ArrayList, HashSet, LinkedList, HashMap.
  • Algorithms: Các phương thức thao tác với collections như sắp xếp, tìm kiếm, đảo ngược, v.v.
  • Iterator: Cung cấp cách thức duyệt qua các phần tử của một collection.

2. Các loại Collection trong Java Link to heading

a) List Link to heading

  • List là một tập hợp các phần tử có thứ tự (ordered) và có thể chứa các phần tử trùng lặp. Các phần tử trong List có thể truy cập theo chỉ số (index).

    Các lớp phổ biến cài đặt List:

    • ArrayList: Cung cấp khả năng truy cập nhanh và có thể thay đổi kích thước.
    • LinkedList: Cung cấp khả năng chèn và xóa nhanh nhưng truy cập chậm hơn so với ArrayList.

    Ví dụ:

    import java.util.ArrayList;
import java.util.List;

public class Example {
    public static void main(String[] args) {
        List<String> list = new ArrayList<>();
        list.add("Java");
        list.add("Python");
        list.add("C++");

        // Truy cập phần tử
        System.out.println(list.get(0));  // In "Java"
    }
}

b) Set Link to heading

Set là một tập hợp các phần tử không có thứ tự và không có phần tử nào bị trùng lặp. Set tự động loại bỏ các phần tử trùng nhau.

Các lớp phổ biến cài đặt Set:

HashSet: Không duy trì thứ tự và không cho phép phần tử trùng lặp. LinkedHashSet: Duy trì thứ tự chèn vào của các phần tử. TreeSet: Sắp xếp các phần tử theo thứ tự tự nhiên hoặc theo một Comparator. Ví dụ:

Sao chép mã
import java.util.HashSet;
import java.util.Set;

public class Example {
    public static void main(String[] args) {
        Set<String> set = new HashSet<>();
        set.add("Java");
        set.add("Python");
        set.add("Java");  // Phần tử "Java" sẽ không được thêm vào

        // In tất cả các phần tử
        System.out.println(set);  // In [Java, Python]
    }
}

c) Map Link to heading

Map là một cấu trúc dữ liệu lưu trữ các cặp khóa-giá trị (key-value). Mỗi khóa trong Map là duy nhất và ánh xạ tới một giá trị.

  • Các lớp phổ biến cài đặt Map:

    • HashMap: Không duy trì thứ tự của các cặp khóa-giá trị và không cho phép phần tử trùng lặp ở khóa.

    • LinkedHashMap: Duy trì thứ tự chèn vào của các cặp khóa-giá trị.

    • TreeMap: Sắp xếp các cặp khóa-giá trị theo thứ tự tự nhiên của khóa hoặc theo Comparator. Ví dụ:

Sao chép mã
import java.util.HashMap;
import java.util.Map;

public class Example {
    public static void main(String[] args) {
        Map<String, Integer> map = new HashMap<>();
        map.put("Java", 10);
        map.put("Python", 5);
        map.put("C++", 8);

        // Truy cập giá trị theo khóa
        System.out.println(map.get("Java"));  // In 10
    }
}

d) Queue Queue là một tập hợp các phần tử được tổ chức theo nguyên tắc FIFO (First In, First Out). Các lớp cài đặt phổ biến của Queue bao gồm LinkedList, PriorityQueue.

Ví dụ:

Sao chép mã
import java.util.LinkedList;
import java.util.Queue;

public class Example {
    public static void main(String[] args) {
        Queue<String> queue = new LinkedList<>();
        queue.add("Java");
        queue.add("Python");
        queue.add("C++");

        // Lấy phần tử khỏi hàng đợi
        System.out.println(queue.poll());  // In "Java"
    }
}

3. Các phương thức cơ bản trong Collections Link to heading

  • add(E e): Thêm phần tử vào collection.
  • remove(Object o): Xóa phần tử khỏi collection.
  • contains(Object o): Kiểm tra xem collection có chứa phần tử cụ thể hay không.
  • size(): Trả về số lượng phần tử trong collection.
  • isEmpty(): Kiểm tra xem collection có rỗng hay không. clear(): Xóa tất cả phần tử khỏi collection.

4. Sử dụng Iterator Link to heading

Iterator là một interface giúp duyệt qua các phần tử trong một collection mà không cần phải biết trước loại collection.

Ví dụ:

Sao chép mã
import java.util.ArrayList;
import java.util.Iterator;
import java.util.List;

public class Example {
    public static void main(String[] args) {
        List<String> list = new ArrayList<>();
        list.add("Java");
        list.add("Python");
        list.add("C++");

        Iterator<String> iterator = list.iterator();
        while (iterator.hasNext()) {
            System.out.println(iterator.next());
        }
    }
}

5. Thuật toán trong Java Collections Link to heading

Java Collections Framework cung cấp một số thuật toán hữu ích như:

Sorting (Sắp xếp): Có thể sử dụng phương thức Collections.sort() để sắp xếp các phần tử trong List.

Ví dụ:

Sao chép mã
import java.util.Collections;
import java.util.List;
import java.util.ArrayList;

public class Example {
    public static void main(String[] args) {
        List<String> list = new ArrayList<>();
        list.add("Java");
        list.add("Python");
        list.add("C++");

        Collections.sort(list);  // Sắp xếp theo thứ tự alphabet

        System.out.println(list);  // In [C++, Java, Python]
    }
}
