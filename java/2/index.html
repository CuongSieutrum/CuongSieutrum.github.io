1. Giới thiệu về OOP Link to heading

Lập trình hướng đối tượng (Object-Oriented Programming - OOP) trong Java là một phương pháp lập trình tổ chức mã nguồn thành các đối tượng, với mục tiêu làm cho việc phát triển, bảo trì và mở rộng ứng dụng trở nên dễ dàng hơn. Java hỗ trợ đầy đủ các nguyên lý của OOP. Các nguyên lý chính trong OOP bao gồm:

Lớp (Class) Link to heading

Lớp là một bản thiết kế cho các đối tượng. Nó định nghĩa các thuộc tính (fields) và hành vi (methods) mà đối tượng của lớp đó sẽ có.

Ví dụ: Chương trình Java cơ bản Link to heading 

class Car {
    // Thuộc tính (fields)
    String brand;
    String model;
    int year;
    
    // Hành vi (method)
    void start() {
        System.out.println("Car is starting");
    }
}

2. Đối tượng (Object) Link to heading

Kế thừa cho phép một lớp con (subclass) kế thừa các thuộc tính và phương thức của lớp cha (superclass). Điều này giúp giảm thiểu mã nguồn lặp lại và tăng tính tái sử dụng.

Ví dụ: Chương trình Java cơ bản Link to heading 

class Animal {
    void sound() {
        System.out.println("Some sound");
    }
}

class Dog extends Animal {
    void sound() {
        System.out.println("Bark");
    }
}

public class Main {
    public static void main(String[] args) {
        Dog dog = new Dog();
        dog.sound(); // In ra: Bark
    }
}

4. Đa hình (Polymorphism) Link to heading

Đa hình là khả năng của một phương thức hoặc đối tượng có thể có nhiều hình thức khác nhau. Trong Java, đa hình có thể được thể hiện qua overloading (nạp chồng phương thức) và overriding (ghi đè phương thức).

Overloading (nạp chồng phương thức): Là khi nhiều phương thức có cùng tên nhưng khác tham số (số lượng hoặc kiểu dữ liệu).

Ví dụ: Chương trình Java cơ bản Link to heading 

class Calculator {
    int add(int a, int b) {
        return a + b;
    }

    double add(double a, double b) {
        return a + b;
    }
}

Overriding (ghi đè phương thức): Là khi lớp con định nghĩa lại phương thức của lớp cha với cùng tên và cùng kiểu trả về.

Ví dụ: Chương trình Java cơ bản Link to heading 

class Animal {
    void sound() {
        System.out.println("Some sound");
    }
}

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("Bark");
    }
}

5. Đóng gói (Encapsulation) Link to heading

Đóng gói là quá trình che giấu thông tin và chỉ cho phép truy cập vào dữ liệu qua các phương thức getter và setter. Điều này giúp bảo vệ dữ liệu khỏi sự truy cập trực tiếp và các thay đổi không mong muốn.

Ví dụ: Chương trình Java cơ bản Link to heading 

class Person {
    private String name; // Thuộc tính private

    // Getter
    public String getName() {
        return name;
    }

    // Setter
    public void setName(String name) {
        this.name = name;
    }
}

public class Main {
    public static void main(String[] args) {
        Person person = new Person();
        person.setName("John");
        System.out.println(person.getName());
    }
}

6. Trừu tượng (Abstraction) Link to heading

Trừu tượng giúp ẩn đi các chi tiết phức tạp và chỉ cung cấp các thông tin cần thiết cho người sử dụng. Điều này có thể đạt được thông qua lớp trừu tượng (abstract class) và giao diện (interface).

Lớp trừu tượng (Abstract Class): Lớp này không thể tạo đối tượng trực tiếp và có thể chứa phương thức abstract (không có phần thân).

Ví dụ: Chương trình Java cơ bản Link to heading 

abstract class Animal {
    abstract void sound();
}

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("Bark");
    }
}

Giao diện (Interface): Là một tập hợp các phương thức mà lớp phải cài đặt.

interface Animal {
    void sound();
}

class Dog implements Animal {
    @Override
    public void sound() {
        System.out.println("Bark");
    }
}
